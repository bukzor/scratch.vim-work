# Principles

Guiding constraints for the modernized vim config. Each file is one principle
with rationale.

## What Belongs

Durable commitments that shape all downstream decisions: philosophical stances,
concrete constraints, quality properties.

Examples: "prefer builtins over plugins", "must work over SSH", "every line understood"

## What Does NOT Belong

- Functional needs (→ needs.kb/)
- Tool evaluations (→ needs.kb/{need}.kb/)
- Decisions with alternatives (→ docs/dev/adr/)
