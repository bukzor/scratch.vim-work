# Demand-Driven Curation

Don't migrate everything forward. Start clean, promote only what you actually
miss.

This applies everywhere: survival kit contents, plugin selection, keymap
migration, the vimrc.d/ fossil-record triage. The existing config is a quarry
to mine on demand, not a checklist to carry forward.

An item earns its place by being missed in practice, not by existing in the
old config. This converts maintenance from exhaustive review into
demand-driven curation — only proven-valuable items survive.
