# Design Principles

Derived principles that bridge constitutional values to concrete technical
choices. Each principle is a reusable reasoning step — it should be traceable
back to one or more values in 000, and invoked by one or more decisions in 050.

## What Belongs

Claims of the form "because [value], we should [principle]." Reusable across
multiple concrete decisions.

## What Does NOT Belong

- Input facts or core values (→ 000-constitutional-values.kb/)
- Concrete choices between specific alternatives (→ 050-technical-decisions.kb/)
