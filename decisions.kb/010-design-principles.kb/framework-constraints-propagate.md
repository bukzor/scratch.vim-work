# Framework Constraints Propagate Harder Than Leaf Constraints

A bad plugin can be wrapped, forked, or dropped. A bad framework shapes every
module, every load path, and every debugging session.

When choosing infrastructure, prefer the option that imposes the fewest
constraints on everything above it. A thin orchestrator you control is more
durable than a rich framework you don't.
