# Mentoring and Onboarding Is a Use Case

When mentoring or onboarding others, a key workflow is: "grab this one file from
github.com/bukzor/dotfiles and it Just Works to fix your problem."

This means individual config files (and ideally small groups of files/directories)
need to function with minimal environment assumptions. The config isn't purely
solo — it's also a library of transferrable pieces.
